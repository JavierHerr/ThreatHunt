Report Information

Hunt Title: Azuki Series: Pt 1: Port of Entry
Date: March 11, 2026
Analyst: Javier Herrera
Case ID: TH-001
Classification: Internal
Status: Completed

Executive Summary

Following the discovery of company supplier contracts and pricing data on underground forums, a threat hunt was initiated to determine the source and scope of the compromise. Analysis identified a multi-stage intrusion that began with malicious payload delivery from an external IP address, followed by hidden PowerShell execution, credential dumping, lateral movement via RDP, and likely data exfiltration. The attacker used built-in Windows tools such as PowerShell, certutil, cmdkey, mstsc, net, and curl to blend into normal system activity while staging payloads, harvesting credentials, and moving to additional systems. Evidence also indicates that stolen data was collected into an archive and uploaded to a Discord webhook, which is believed to be the mechanism used to exfiltrate sensitive supplier contract and pricing information later found on underground forums. The findings confirm unauthorized access, credential theft, lateral movement, and probable data exfiltration within the environment.

Objective

Identify evidence of malicious script execution, payload delivery, credential theft, lateral movement, and outbound data exfiltration associated with the compromise.

Hypothesis

Adversaries may be using PowerShell and other built-in Windows tools to download and execute payloads, dump credentials from memory, move laterally across systems using valid accounts, and exfiltrate collected data over common web protocols while evading traditional signature-based detections.

Scope

  • Timeframe: November 19, 2025 – November 20, 2025
  • Assets: 1 Windows endpoint and 1 internal server confirmed affected
  • Users: Corporate user accounts and administrative credentials in production
  • Environment: Enterprise Windows environment
  • Data Sources: EDR, SIEM telemetry, process execution logs, logon telemetry, file activity, and network telemetry

Methodology

  • Reviewed endpoint process telemetry for suspicious PowerShell, certutil, cmdkey, mstsc, curl, and net execution
  • Correlated process activity with file creation, registry activity, and network connections
  • Tracked suspicious account usage and remote interactive logons to identify lateral movement
  • Reconstructed the likely attack timeline from payload delivery through exfiltration
  • Mapped observed behavior to MITRE ATT&CK techniques for credential access, remote access, and exfiltration

Key Findings

Finding 1: Malicious Payload Delivery and Script Execution

Severity: Critical

Confidence: High

Affected Asset: Initial compromised Windows endpoint

PowerShell was used with -ExecutionPolicy Bypass and, in some cases, -WindowStyle Hidden to download and execute files from 78.141.196.6:8080. Observed artifacts included wupdate.ps1 and wupdate.bat saved to the user's Temp directory. This behavior is consistent with malware staging and execution using built-in scripting tools.

Assessment: Confirmed malicious payload delivery and execution.

Status: Confirmed.

Finding 2: Secondary Tool Transfer via certutil

Severity: Critical

Confidence: High

Affected Asset: Initial compromised Windows endpoint

The attacker used certutil.exe -urlcache -f to download a file named AdobeGC.exe from 78.141.196.6:8080 and saved it locally as C:\ProgramData\WindowsCache\mm.exe. The renaming and staging location indicate deliberate use of a trusted Windows binary to retrieve a malicious payload.

Assessment: Living-off-the-land technique used for malicious tool delivery.

Status: Confirmed.

Finding 3: Credential Dumping via Mimikatz-like Activity

Severity: Critical

Confidence: High

Affected Asset: Initial compromised Windows endpoint

The command "mm.exe" privilege::debug sekurlsa::logonpasswords exit was observed, strongly indicating Mimikatz-style credential dumping from LSASS memory. This would have enabled the attacker to collect plaintext passwords, NTLM hashes, and Kerberos material for additional access.

Assessment: Confirmed credential theft activity.

Status: Confirmed.

Finding 4: Credential Storage and Lateral Movement Preparation

Severity: High

Confidence: High

Affected Asset: Initial compromised Windows endpoint

The attacker used cmdkey.exe /generic:10.1.0.188 /user:fileadmin /pass:********** to store credentials for the internal host 10.1.0.188. This is a common precursor to remote authentication and lateral movement using stolen or compromised credentials.

Assessment: Strong evidence of preparation for lateral movement.

Status: Confirmed.

Finding 5: Lateral Movement via Remote Desktop

Severity: Critical

Confidence: High

Affected Asset: Internal server 10.1.0.188

The command mstsc.exe /v:10.1.0.188 shows the attacker initiated an RDP session to the internal host 10.1.0.188. In context with the preceding cmdkey.exe credential storage activity, this indicates successful or attempted lateral movement into another system.

Assessment: Confirmed remote access attempt consistent with attacker pivoting.

Status: Confirmed.

Finding 6: Privilege Escalation / Account Manipulation

Severity: High

Confidence: Moderate

Affected Asset: Compromised systems under attacker control

The command net.exe localgroup Administrators support was observed in the breach sequence. While the captured command did not include /add, the context suggests investigation into or attempted manipulation of local administrator group membership for the support account.

Assessment: Suspicious behavior consistent with privilege escalation or persistence.

Status: Investigate further.

Finding 7: Data Staging and Exfiltration to Discord

Severity: Critical

Confidence: High

Affected Asset: Compromised endpoint

The attacker used curl.exe to upload C:\ProgramData\WindowsCache\export-data.zip to a Discord webhook. The file name, staging location, and outbound upload pattern strongly indicate collection and exfiltration of data from the victim host to attacker-controlled infrastructure.

Assessment: Confirmed probable data exfiltration.

Status: Confirmed.

Attack Timeline

  • PowerShell downloaded wupdate.ps1 and wupdate.bat from 78.141.196.6:8080
  • Downloaded scripts were executed with hidden windows and execution policy bypass
  • certutil.exe retrieved an additional payload and saved it as mm.exe
  • mm.exe was executed with Mimikatz-style credential dumping arguments
  • cmdkey.exe stored credentials for 10.1.0.188
  • mstsc.exe was used to open an RDP session to 10.1.0.188
  • net.exe activity suggested possible administrator group interaction involving support
  • Data was staged into export-data.zip and uploaded to a Discord webhook using curl.exe

Indicators of Compromise

  • External IP: 78.141.196.6
  • Internal Target: 10.1.0.188
  • Files: wupdate.ps1, wupdate.bat, mm.exe, export-data.zip
  • Paths: C:\Users\KENJI~1.SAT\AppData\Local\Temp\, C:\ProgramData\WindowsCache\
  • Accounts: fileadmin, support
  • Utilities Used: powershell.exe, certutil.exe, cmdkey.exe, mstsc.exe, net.exe, curl.exe
  • Exfiltration Channel: Discord webhook

MITRE ATT&CK Mapping

  • T1059.001: PowerShell
  • T1105: Ingress Tool Transfer
  • T1218: Signed Binary Proxy Execution / LOLBins
  • T1003.001: OS Credential Dumping: LSASS Memory
  • T1078: Valid Accounts
  • T1021.001: Remote Services: Remote Desktop Protocol
  • T1098: Account Manipulation
  • T1567: Exfiltration Over Web Service

Impact Assessment

Risk Level: Critical

The investigation identified confirmed malicious execution, credential dumping, likely compromise of valid accounts, lateral movement to at least one additional host, and probable exfiltration of sensitive data. Based on the observed behavior, the environment should be treated as actively compromised, with potential impact to authentication systems, internal servers, and confidential business information.

Actions Taken

  • Reviewed suspicious endpoint and server telemetry
  • Identified malicious use of PowerShell, certutil, cmdkey, mstsc, net, and curl
  • Tracked attacker movement from the initial host to internal system 10.1.0.188
  • Documented indicators including IPs, files, accounts, and exfiltration method
  • Confirmed likely data theft via Discord webhook upload

Recommendations

  • Immediately isolate affected endpoints and the internal host 10.1.0.188
  • Reset passwords for impacted accounts, especially fileadmin, support, and any accounts exposed to LSASS dumping
  • Block communications to 78.141.196.6 and the identified Discord webhook infrastructure
  • Review and restrict RDP access, especially for privileged accounts
  • Enable and retain PowerShell Script Block Logging and full command-line logging
  • Hunt across the enterprise for wupdate.ps1, wupdate.bat, mm.exe, and export-data.zip
  • Review local administrator group membership and remove unauthorized accounts
  • Develop or tune detections for credential dumping, LOLBin abuse, Discord webhook uploads, and suspicious PowerShell download activity

Limitations

  • Some command context was inferred from observed process arguments rather than full forensic images
  • Not all endpoint history outside the defined timeframe was available for review
  • The exact contents of export-data.zip were not validated in the provided evidence set
  • Account manipulation involving support requires additional confirmation from group membership and account creation telemetry

Conclusion

The threat hunt confirmed a serious compromise involving malicious payload delivery, hidden PowerShell execution, credential dumping, lateral movement through RDP, and probable data exfiltration. The observed sequence reflects a hands-on-keyboard intrusion using legitimate Windows utilities to reduce detection and expand access. Immediate containment, credential rotation, and enterprise-wide hunting are recommended to prevent further attacker activity and identify any additional affected systems.
